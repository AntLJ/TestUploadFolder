【CheckBaseRepExist 使用方法】	2007/08/29 H.Hyodo

■モジュール名

	CheckBaseRepExist

■機能

	・	行政界ポリゴン（City_Admin）に対し、大字・町丁目代表点（Base_Rep_Point）が
		整備されているかどうかチェックする
	・	大字・町丁目代表点（Base_Rep_Point）について、大字／字間の階層切れがないかどうか
		（例：「xxxxxxxx101」に対し「xxxxxxxx000」が整備されているかどうか）をチェックする。
	・	CS_Addr_Point について、上位11桁の Base_Rep_Point が整備されているかどうかを
		チェックする。

■実行形式

	CheckBaseRepExist [オプション]

■オプション

	-ca
		全市区町村についてチェックする
		（市区町村リストは行政界ポリゴンから自動生成）

	-cs <citycode>
		指定した市区町村コードのエリアについてチェックを実施

	-cl <filename>
		チェック対象の市区町村コードリストファイルを指定

	-o <filename>
		ログ出力先ファイル指定（必須）

	-s <filename>
		ステータス出力先ファイル指定（必須）

	※ -c* オプションはいずれかを指定する必要あり。

■環境変数

	DB_ADDRESS
		GeoDB接続文字列（住所系サーバ）

	FC_GOUPOINT
		号ポイントのフィーチャクラス名

	FC_CITYADMIN
		行政界ポリゴンのフィーチャクラス名

	FC_BASEREP
		大字・町丁目代表点のフィーチャクラス名

	FC_PAREA
		パレアポイントのフィーチャクラス名

	FC_CSADDRPOINT
		CSポイントのフィーチャクラス名

	JDB_CONNECT
		住所DBへの接続文字列を指定

■エラーログ

	ERROR	H04xx001	Base_Rep_Point がない
		→	当該コードを持つ行政界に対して Base_Rep_Point や号ポイントが
			整備されていない
		→	CSポイントに対して上位11桁の Base_Rep_Point が整備されていない

	ERROR	H04xx002	大字代表点がない
		→	大字代表コード（住所コード８桁＋「000」）に対し、ポイントが
			整備されていない（行政界の存在する８桁について）
		→	字の代表点（字・丁目コード：「101〜850」or「901〜999」）が
			整備されているのに大字代表点が整備されていない

// PAREAのエラーは廃止
//	ERROR	H04xx003	PAREA のみ
//		→	当該コードを持つポイントが PAREA しかない

	ERROR	H04xx004	廃止 or 無効な11桁
		→	当該コードが住所マスタ上で廃止扱い、または住所マスタ上にない

	WARNING	H04xx005	号ポイントより生成
		→	Base_Rep_Point は整備されていないが、号ポイントは存在する

	※3.2.0.4から、ADDRCODE2のチェックが追加され、ADDRCODE2のチェック結果についてはログの後ろに(ADDRCODE2)と追加されます。
